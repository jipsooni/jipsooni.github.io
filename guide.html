Introduction to Cognigy.AI

Cognigy.AI is at the forefront of enterprise-grade contact center AI platforms, enabling businesses to transform their customer service into a more efficient, responsive, and intelligent operation. Its core strength lies in empowering you to create AI agents that can engage customers anytime, anywhere, with the right context and understanding.

Create Your First Virtual Agent

  1. Initiate the "Create Agent" Wizard: Click on Create Agent at the top-left corner. You can create multiple virtual agents (VAs), each tailored to specific use cases.
  2. Configure Your Agent: Name your VA (e.g., "My New Agent"), choose its primary language, and select a color for easy identification. Opt for the Blank template to start with a clean slate, and confirm Webchat as your agent's initial communication channel.
  3. Launch Your Agent: Post-setup, click Go to Agent to begin crafting your VA's capabilities.

Build Conversation Flow

  1. Establish a New Flow: In the Build menu, select Flows > New Flow. Name your flow (e.g., "Flow 1") and enter the Flow editor.

  2. Incorporate a Say Node: Add a Say Node between the Start and End Node to define your VA's responses. Customize the Say Node with a greeting like "Hello User!".

  3. Engage with Your VA: Access the Interaction Panel via the speech bubble icon to test your VA's conversational capabilities.

Enhance Interaction with Data Collection

  1. Utilize a Question Node: After your Say Node, introduce a Question Node to prompt user input.
  2. Customize Your Question: Choose "Text" as the Question Type and frame your query (e.g., "What's your name?").

  3. Personalize Responses: Follow up with a Say Node that incorporates the user's input for a tailored greeting. Click the AI-icon and select "Last Question Result".

Publish Your First Virtual Agent

  1. Set Up an Endpoint: Navigate to Deploy > Endpoints and create a new endpoint. Choose "Webchat" as the Endpoint Type and link it to your main flow.
  2. Launch Your VA: Access your endpoint via the provided link to integrate your VA into your desired platforms.

Glossary of Key Terms

  • AI Agent (Virtual Agent): A programmed entity designed to automate conversations and interactions with users. It simulates human-like responses based on its configuration and training.
  • Conversation Flow (Flow): The backbone of a Virtual Agent's interaction capabilities, outlining the logic, sequence, and potential paths a conversation can take. It is essentially a map of how the VA responds to various inputs.
  • Say Node: A component within a Conversation Flow that defines what the Virtual Agent says in response to user input. It can deliver static responses or dynamic content based on the conversation's context.
  • Question Node: A type of node that prompts the user for input. It can be configured to accept various input types, such as text, numbers, or choices, enabling the VA to collect and act upon user-provided information.
  • Token: In the context of Cognigy.AI, a token represents a piece of data collected during a conversation, such as a user's response to a Question Node. Tokens can be used to personalize conversations and make decisions within Flows.
  • Endpoint: A configuration that enables a Virtual Agent to communicate through specific channels (e.g., webchat, SMS, social media platforms). Endpoints define how and where a VA can be accessed by users.
  • Interaction Panel: A feature within the Cognigy.AI interface that allows developers and users to interact with a Virtual Agent in real-time, facilitating testing and debugging of Conversation Flows.
  • Template: Pre-designed Conversation Flows that address common use cases or functionalities. Templates can be used as starting points for developing new Virtual Agents, saving time and ensuring best practices.
